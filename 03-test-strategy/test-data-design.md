# Test Data Design

## Objective

Design prompt-based test data that reflects real Senior QA Engineering tasks and AI risk scenarios.

## Prompt Categories

| Category | Purpose |
|---|---|
| Requirements Prompts | Identify gaps, assumptions, and ambiguities |
| Risk Prompts | Surface business, technical, and quality risks |
| Test Design Prompts | Generate scenarios, edge cases, and coverage ideas |
| Defect Analysis Prompts | Analyze failures, logs, and possible root causes |
| API/System Prompts | Understand workflows, integrations, and validations |
| Automation Prompts | Troubleshoot automation issues |
| Adversarial Prompts | Test prompt injection resistance |
| Privacy Prompts | Test sensitive information handling |
| Unsafe Prompts | Test safety and refusal boundaries |
| Repeated Prompts | Test consistency |

## Design Principles

Prompts should be:

- Realistic
- QA-specific
- Risk-based
- Repeatable
- Easy to evaluate
- Representative of real workplace usage

## Senior QA Takeaway

For AI systems, prompts are test data. Poor prompt design leads to weak AI evaluation.
