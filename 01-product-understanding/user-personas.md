# Persona 1
## QA Engineer

## Overview

A QA Engineer uses ChatGPT to improve productivity, accelerate analysis, and support quality-related decision-making throughout the software development lifecycle.

## Goals

* Improve testing efficiency
* Increase test coverage
* Accelerate defect investigation
* Enhance quality documentation
* Learn new domains and technologies faster

## Common Use Cases

1. Analyze requirements to identify ambiguities, gaps, and missing acceptance criteria.
2. Generate functional, negative, boundary, and exploratory test scenarios.
3. Identify risks and support risk-based testing.
4. Troubleshoot automation failures and review test scripts.
5. Understand APIs and generate API testing scenarios.
6. Analyze logs, investigate defects, and explore possible root causes.
7. Generate test data and edge-case inputs.
8. Draft defect reports, test plans, and test summaries.
9. Learn unfamiliar business domains, systems, and technologies.

## Success Criteria

* Faster requirement analysis
* Improved test coverage
* Quicker defect investigation
* Better risk identification
* Higher quality documentation

## Key Risks

* Inaccurate or incomplete responses
* Missing business-critical scenarios
* Lack of domain-specific context
* Over-reliance on AI-generated outputs
